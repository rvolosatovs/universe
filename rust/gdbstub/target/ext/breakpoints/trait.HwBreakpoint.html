Expand description

Nested Target Extension - Set/Remove Hardware Breakpoints.

See this stackoverflow discussion about the differences between hardware and software breakpoints.

Recommendation: If you’re implementing Target for an emulator that’s using an interpreted CPU (as opposed to a JIT), there shouldn’t be any reason to implement this extension (as software breakpoints are likely to be just-as-fast).
